About This Example

This is a simple example of passwordless authentication using magic links. The authentication flow works as follows:

  1. User enters their email address
  2. A magic link is sent to the user's email
  3. User clicks the magic link
  4. User is authenticated and a session is created

The library handles all the security aspects, including:

  • Secure token generation
  • Token expiration
  • Rate limiting
  • Secure session management
